# USB22-485 WinUSB ARCNET Driver

A **user-mode Windows library and driver package** for the Contemporary Controls USB22-485
ARCNET adapter (VID `0x0D0B`, PID `0x1002` / `0x1003`), built entirely on the in-box
`winusb.sys` — no kernel-mode code required.

> **Turkish summary / Türkçe özet:** Contemporary Controls USB22-485 ARCNET adaptörü için
> Windows user-mode WinUSB sürücüsü ve kütüphanesi. Kernel sürücüsü yoktur; Windows'un
> yerleşik `winusb.sys`'ini kullanır. API, C kütüphanesi olarak statik bağlanır.

---

## Background / Motivation

Contemporary Controls' original USB22-485 software ships with a cross-signed **kernel-mode
driver**. Windows updates starting in early 2026 enforce stricter driver signature policies
that block cross-signed third-party kernel drivers on many configurations, rendering the
device unusable without manual workarounds.

This project takes a different approach: it **never touches the kernel**. The INF file in
`driver/` binds Microsoft's own `winusb.sys` to the device, and all ARCNET protocol work
happens in a pure user-mode C library (`src/arcnet.c`) over the standard WinUSB API
(`WinUsb_WritePipe` / `WinUsb_ReadPipe`).

The USB wire protocol was reverse-engineered entirely from USB captures. See
[`reference/usb22-protocol-notes.md`](reference/usb22-protocol-notes.md) for the full
framing documentation (register access, init sequence, transmit/receive paths, event
messages).

## Features

| Capability | Details |
|---|---|
| Device enumeration | `arc_list_devices()` — finds all USB22-485 adapters via SetupAPI |
| Open / close | `arc_open()` / `arc_close()` — allocates/frees per-device context; `arc_close(NULL)` is safe |
| Node initialization | `arc_init()` — full CMD04 → handshake → COM20022 config sequence (~2.5 s) |
| Register access | `arc_register()` — read or write any COM20022 chip register (0–7) |
| Transmit | `arc_transmit()` — sends an ARCNET frame; optional ACK detection via TMA bit |
| Receive | `arc_receive()` — non-blocking poll for incoming frames |
| Event read | `arc_read_event()` — reads unsolicited status/RECON events from EP 0x81 |
| ACK detection | Polls COM20022 status register 0 bit 1 (TMA — Transmit Message Acknowledged) for up to 100 ms post-transmit |
| Device-gone detection | Fatal USB errors set an internal flag; `arc_reopen()` re-establishes the session |
| Transient busy separation | `ARC_ERR_NET_BUSY` (ARCNET RECON / GLE 121) vs `ARC_ERR_IO` (real hardware error) |
| Multi-device | Each `arc_open()` returns an independent context; N adapters run in parallel |
| Thread safety | Per-context `CRITICAL_SECTION` (recursive); concurrent calls on the same context serialize; different contexts are fully parallel |
| Leveled logging | `ARC_LOG_NONE` (default) / `ERROR` / `INFO` / `DEBUG`; optional callback for log redirection |
| Management GUI | WPF (.NET 8) tool — installs driver, lists devices, runs tests, shows colored log |

## Driver Installation

Full instructions are in [`driver/INSTALL.md`](driver/INSTALL.md). Quick summary:

### Method A — Recommended (works with Secure Boot ON)

Run an **Administrator PowerShell** from the `driver/` directory:

```powershell
.\sign.ps1
```

`sign.ps1` creates a machine-local self-signed code-signing certificate, registers it in
the **Trusted Root CA** and **Trusted Publisher** certificate stores, and signs
`usb22_winusb.cat`. Then install the driver package:

```cmd
pnputil /add-driver "driver\usb22_winusb.inf" /install
```

Or right-click `driver\install.bat` → **Run as administrator**.

Plug in the USB22-485. After ~2 s the device transitions from its bootloader PID
(`0xB001`/`0xB002`) to the operational PID (`0x1002`/`0x1003`) and the driver binds
automatically.

### Method B — bcdedit test signing (Secure Boot must be OFF)

```cmd
bcdedit /set testsigning on   :: then reboot
pnputil /add-driver "driver\usb22_winusb.inf" /install
```

### GUI installation (one-click)

Launch `gui\bin\Debug\net8.0-windows\WetrechArcnetManager.exe` and click
**⬇ WinUSB Sürücüsünü Kur**. The GUI runs `sign.ps1` then `pnputil` and reports each
step in a resizable, color-coded log panel.

> **BitLocker note:** Do not use Shift+Restart → "Startup Settings" to disable signature
> enforcement on BitLocker-protected machines — it will trigger BitLocker recovery.
> Use Method A (`sign.ps1`) instead; it works without any reboot and is Secure Boot
> compatible.

> **Self-signed scope:** The certificate created by `sign.ps1` is valid only on the
> machine where it was generated. For broad distribution, sign with a commercial EV
> code-signing certificate (see `driver/INSTALL.md` for the roadmap).

## API Reference

Include `src/arcnet.h` and link `arcnet.lib`, `setupapi.lib`, `winusb.lib`.

### Result codes

| Code | Value | Meaning |
|---|---|---|
| `ARC_OK` | `0` | Success; if `waitAck=true` in `arc_transmit`, the packet was acknowledged |
| `ARC_NO_PACKET` | `1` | `arc_receive` / `arc_read_event`: channel empty, call again |
| `ARC_NOT_ACKED` | `2` | `arc_transmit`: packet sent but TMA bit not set within 100 ms |
| `ARC_ERR_OPEN` | `-1` | Device not found or `CreateFile` failed |
| `ARC_ERR_IO` | `-2` | USB read/write failure (transient hardware error) |
| `ARC_ERR_TIMEOUT` | `-3` | Response budget exhausted waiting for device reply |
| `ARC_ERR_PARAM` | `-4` | Invalid argument (e.g. NULL data pointer, zero length) |
| `ARC_ERR_ECHO` | `-5` | Device response opcode / register echo mismatch |
| `ARC_ERR_DEVICE_GONE` | `-6` | Device physically removed; call `arc_reopen()` then `arc_init()` |
| `ARC_ERR_NET_BUSY` | `-7` | ARCNET RECON in progress / network busy (GLE 121); transient, retry |

```c
const char *arc_result_str(arc_result_t r);
```
Returns a short human-readable string for any `arc_result_t` value.

### Functions

#### `arc_list_devices`
```c
int arc_list_devices(char paths[][256], int maxDevices);
```
Enumerates all connected USB22-485 adapters using SetupAPI. Fills `paths[]` with WinUSB
interface paths (up to `maxDevices`). Returns the number of devices found (0 = none).

---

#### `arc_open`
```c
arc_ctx_t *arc_open(const char *devicePath, bool verbose);
```
Opens a device and allocates an independent context. Pass `devicePath = NULL` to
auto-select the first detected device, or pass a path from `arc_list_devices()`.
`verbose` is a legacy flag; prefer `arc_set_log_level()` after opening.
Returns a non-NULL context on success; returns NULL on failure. The caller must eventually
call `arc_close()` on the returned context.

---

#### `arc_close`
```c
void arc_close(arc_ctx_t *ctx);
```
Closes the USB session, releases all OS handles, and frees the context. Safe to call with
`NULL`. The pointer is invalid after this call. Do not call while another thread is still
inside any API function on the same context.

---

#### `arc_init`
```c
arc_result_t arc_init(arc_ctx_t *ctx, uint8_t nodeID, uint8_t timeout,
                      uint8_t clockPrescaler, bool recvBroadcasts);
```
Runs the full ARCNET node startup sequence (see *Architecture* below). Takes approximately
2.5 seconds on real hardware. Must be called after `arc_open()` (and again after
`arc_reopen()`).

- `nodeID` — this node's ARCNET address (0x01–0xFE)
- `timeout` — reconfiguration timeout byte (`0x18` recommended)
- `clockPrescaler` — COM20022 clock divisor (`0x00` = default speed)
- `recvBroadcasts` — if `true`, the node also receives broadcast frames (destination = 0)

---

#### `arc_register`
```c
arc_result_t arc_register(arc_ctx_t *ctx, bool bWrite, uint8_t reg, uint8_t *value);
```
Reads or writes a single COM20022 chip register (0–7).
`bWrite=false`: `*value` is filled with the register's current value.
`bWrite=true`: `*value` is written to the register.

---

#### `arc_transmit`
```c
arc_result_t arc_transmit(arc_ctx_t *ctx, uint8_t destNode,
                          const uint8_t *data, int len, bool waitAck);
```
Sends an ARCNET frame. `len` must be 1–252 bytes.

`waitAck=true`: after writing to the USB TX pipe, polls COM20022 status register 0
bit 1 (TMA — Transmit Message Acknowledged) for up to 100 ms (`ARC_ACK_POLL_BUDGET_MS`).
Returns `ARC_OK` if the bit is set (packet acknowledged by the destination node) or
`ARC_NOT_ACKED` if the budget expires. On a network with a single device, `ARC_NOT_ACKED`
is the normal outcome.

`waitAck=false`: returns `ARC_OK` as soon as the USB write succeeds, without waiting for
delivery confirmation (higher throughput).

---

#### `arc_receive`
```c
arc_result_t arc_receive(arc_ctx_t *ctx, uint8_t *src, uint8_t *dst,
                         uint8_t *data, int *len);
```
Non-blocking poll for an incoming ARCNET frame. Times out after 150 ms
(`ARC_RECEIVE_TIMEOUT_MS`) and returns `ARC_NO_PACKET` if nothing arrived.
`data[]` must be at least 253 bytes. On `ARC_OK`, `*src`, `*dst`, `*data`, and `*len`
are populated.

---

#### `arc_read_event`
```c
arc_result_t arc_read_event(arc_ctx_t *ctx, uint8_t *buf, int bufsize,
                             uint32_t timeout_ms, int *out_len);
```
Reads one raw event/status packet from EP 0x81 (unsolicited device → host, e.g. RECON
events). `buf` should be at least `ARC_EP_EVT_MAXPACKET` (64) bytes.
If `timeout_ms` is 0 the current pipe timeout policy is kept. On `ARC_OK`, `*out_len`
contains the number of bytes received.

---

#### `arc_reopen`
```c
arc_result_t arc_reopen(arc_ctx_t *ctx);
```
Attempts to re-establish the WinUSB session after the device was physically removed
(detected via `ARC_ERR_DEVICE_GONE`). Closes any dead handles, clears the internal
`device_gone` flag, and tries to open the same device path that was used by `arc_open()`.
If the device is still absent, returns `ARC_ERR_DEVICE_GONE` — safe to call again later.
On `ARC_OK` the handles are valid but **the ARCNET node is not initialized**; the caller
must call `arc_init()` before transmitting or receiving.

---

#### Logging controls

```c
void arc_set_log_level(arc_ctx_t *ctx, arc_log_level_t level);
```
Sets the log verbosity for this context. Default is `ARC_LOG_NONE` (silent).

```c
typedef void (*arc_log_fn)(arc_log_level_t level, const char *msg, void *user);
void arc_set_log_callback(arc_ctx_t *ctx, arc_log_fn fn, void *user);
```
Redirects all log output to a caller-supplied callback instead of `stderr`. All messages
include the `[arcnet] ` prefix and a trailing `\n`. Pass `fn = NULL` to restore the
default (`stderr`) output.

```c
void arc_set_verbose(arc_ctx_t *ctx, bool enable);
```
Backward-compatible shorthand: `enable=true` sets `ARC_LOG_DEBUG`, `enable=false` sets
`ARC_LOG_NONE`.

## Usage Example

```c
#include "arcnet.h"
#include <stdio.h>
#include <string.h>

int main(void)
{
    /* 1. Open — NULL auto-selects the first detected USB22-485 */
    arc_ctx_t *ctx = arc_open(NULL, /*verbose=*/false);
    if (!ctx) { fprintf(stderr, "arc_open failed\n"); return 1; }

    /* Optional: enable logging after open */
    arc_set_log_level(ctx, ARC_LOG_INFO);

    /* 2. Initialize the ARCNET node (takes ~2.5 s) */
    arc_result_t r = arc_init(ctx,
                              /*nodeID=*/0x01,
                              /*timeout=*/0x18,
                              /*clockPrescaler=*/0x00,
                              /*recvBroadcasts=*/true);
    if (r != ARC_OK) {
        fprintf(stderr, "arc_init: %s\n", arc_result_str(r));
        arc_close(ctx);
        return 1;
    }

    /* 3. Transmit a packet to node 2, wait for ACK */
    const uint8_t payload[] = "HELLO-ARCNET";
    r = arc_transmit(ctx, /*destNode=*/0x02,
                     payload, (int)strlen((char *)payload),
                     /*waitAck=*/true);
    printf("arc_transmit: %s\n", arc_result_str(r));
    /* ARC_OK = ACKed, ARC_NOT_ACKED = sent but no ACK (normal if no listener),
       ARC_ERR_NET_BUSY = ARCNET RECON in progress (retry later) */

    /* 4. Poll for a received packet */
    uint8_t src, dst, data[253];
    int     data_len = 0;
    r = arc_receive(ctx, &src, &dst, data, &data_len);
    if (r == ARC_OK)
        printf("received %d bytes from node %u\n", data_len, src);
    else if (r == ARC_NO_PACKET)
        printf("nothing received (channel empty)\n");

    /* 5. Close and free */
    arc_close(ctx);
    return 0;
}
```

## Architecture and Design Notes

### Context model

`arc_open()` heap-allocates an `arc_ctx_t` struct and returns it as an opaque pointer.
There is no global state in the library. N adapters can be open simultaneously as N
independent contexts; operations on different contexts never interact.

### Startup sequence

`arc_init()` performs three steps in order, documented in
[`reference/usb22-protocol-notes.md`](reference/usb22-protocol-notes.md):

1. **CMD04** — sends a 2-byte session-start command (`04 00`) to EP 0x01 and reads a
   4-byte echo from EP 0x81 to confirm the USB session is alive.
2. **Handshake** — writes 10 zero bytes to EP 0x02 and reads back 10 bytes to probe the
   data channel.
3. **COM20022 config** — sends a 12-byte initialization command to EP 0x01 configuring
   the ARCNET node ID, timeout, clock prescaler, and broadcast mode. The device takes
   approximately 2.5 seconds to respond (budget: 5 s).

The library waits up to `ARC_BUDGET_INIT_MS` (5000 ms) for the init response.

### ACK detection (TMA bit)

When `arc_transmit()` is called with `waitAck=true`, the library polls COM20022 status
register 0, bit 1 — **TMA (Transmit Message Acknowledged)** — in a loop with 10 ms sleep
intervals for up to 100 ms (`ARC_ACK_POLL_BUDGET_MS`). The TMA bit is set by the COM20022
chip when the destination node has acknowledged the frame. `ARC_NOT_ACKED` means the bit
was never observed within the budget — not a fatal error.

### Device-gone / reconnect

When a WinUSB call returns a fatal error (Windows error codes 6, 22, 31, 433, 1167, 1168)
the library sets an internal `device_gone` flag on the context. All subsequent API calls
on that context immediately return `ARC_ERR_DEVICE_GONE` without touching hardware. Call
`arc_reopen()` to clear the flag and re-establish the session; then call `arc_init()`
again before sending or receiving.

### Transient busy vs. hardware error

`ARC_ERR_NET_BUSY` is returned when `WinUsb_WritePipe` on EP 0x02 fails with
`ERROR_SEM_TIMEOUT` (GLE 121) and `device_gone` is not set. This indicates that the
ARCNET network is undergoing reconfiguration (RECON) and the transmitter is temporarily
unavailable. The appropriate response is to retry after a short delay — the hardware is
not broken.

`ARC_ERR_IO` is returned for other, non-fatal USB pipe errors.

### Thread safety

Each context owns a `CRITICAL_SECTION` initialized at `arc_open()` time and deleted at
`arc_close()` time. Every public API function acquires this lock on entry and releases it
on exit (including all error return paths via `goto done`). The critical section is
recursive (re-entrant), so internal calls — for example, `arc_transmit()` calling
`arc_register()` to read the TMA bit — do not deadlock.

Concurrent calls on **different** contexts are fully parallel (no shared state).
Do not call `arc_close()` while another thread may still be inside any API function on
the same context.

## Limitations and Known Issues

| Item | Detail |
|---|---|
| **Max payload** | ARCNET frames carry at most 252 bytes of payload (`arc_transmit` `len` must be 1–252; `arc_receive` buffer must be ≥ 253 bytes) |
| **Receive is poll-based** | `arc_receive()` blocks for up to 150 ms per call; there is no interrupt-driven or callback-based receive mechanism |
| **ACK poll budget** | The 100 ms TMA poll budget may be insufficient on slow/busy networks; the value (`ARC_ACK_POLL_BUDGET_MS`) is a compile-time constant |
| **No long-frame support** | ARCNET 512-byte "exception" frames are not implemented |
| **Vendor SDK** | The Contemporary Controls vendor SDK (`ArcX.h`, `Talk.exe`) is proprietary and not included in this repository |
| **Self-signed driver** | `sign.ps1` creates a machine-local certificate; the driver package is not suitable for broad distribution without a commercial EV code-signing certificate |
| **Single INF binding** | The driver binds only PID `0x1002` and `0x1003`; the bootloader PIDs (`0xB001`, `0xB002`) are intentionally excluded |
